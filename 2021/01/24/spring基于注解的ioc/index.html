用于创建对象的

@Component:

作用：用于把当前类对象存入spring容器中。

相当于xml配置中的<bean id = "" class = "">

属性：value：用于指定bean的id。当我们不写时，它的默认值是当前类名，且首字母改小写。

同样用于创建对象的：

@Controller：一般用在表现层

@Service：一般用在业务层

@Repository：一般用在持久层

以上三个注解他们的作用和属性与Component是一模一样，这样做有什么好处呢？

是spring框架为我们提供明确的三层使用的注解，可以使我们的三层对象更加清晰。
